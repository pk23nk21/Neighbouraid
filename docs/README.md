# NeighbourAid Documentation

This folder is the long-form companion to the project README. It's meant
to be read in order, but every page can also be picked up on its own.

| File | Read this if you… |
|---|---|
| [01-overview.md](01-overview.md) | are new to NeighbourAid and want the 5-minute tour |
| [02-user-guide.md](02-user-guide.md) | are a reporter or volunteer and want to know how to *use* the app |
| [03-architecture.md](03-architecture.md) | want to understand how the pieces fit together |
| [04-feature-deep-dive.md](04-feature-deep-dive.md) | want to know exactly how a single feature works (AI triage, photos, escalation, …) |
| [05-api-reference.md](05-api-reference.md) | are integrating with the backend or building a client |
| [06-development.md](06-development.md) | are running the project locally for the first time |
| [07-deployment.md](07-deployment.md) | are shipping the app to a server |
| [08-testing.md](08-testing.md) | are writing or running tests |
| [09-troubleshooting.md](09-troubleshooting.md) | hit a problem and need a quick fix |
| [10-glossary.md](10-glossary.md) | see a term and don't know what it means |

The docs assume **no prior familiarity** with the codebase, FastAPI,
React, or Indian crisis-response tooling. Where domain terms come up
they're defined inline or in the glossary.
