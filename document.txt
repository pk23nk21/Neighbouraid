## 1. Executive Summary

NeighbourAid is a full-stack web application that enables people within a locality to post urgent help requests — such as medical emergencies, flood incidents, power outages, or missing persons — and instantly notifies nearby registered volunteers. An AI layer (running locally via Hugging Face models, no external API) classifies the severity of each alert and routes it to the most relevant helpers.

The backend is built in Python (FastAPI), containerized with Docker, and uses GitHub Actions for CI/CD. No paid APIs are used — the entire stack is open source.

## 2. Problem Statement

During crises — whether a medical emergency, a natural disaster, or a missing person — the first few minutes are the most critical. In urban and semi-urban India, people frequently rely on informal WhatsApp groups or word-of-mouth to seek help. These channels are slow, unstructured, and geographically unreliable.

### 2.1 Key Pain Points
- No structured platform to post a crisis and instantly reach nearby volunteers
- Existing platforms (iVolunteer, OpenContribute) are appointment-based, not real-time
- Emergency services (100, 108) are overloaded and often unreachable in time
- No AI-based triage exists to prioritize life-threatening vs. non-urgent requests
- Geo-based volunteer matching is completely absent from current tools

### 2.2 Opportunity
A lightweight, open-source web platform combining real-time crisis posting, geolocation-based volunteer matching, and on-device AI urgency classification can meaningfully reduce emergency response times at the community level — at zero API cost.

## 5. System Architecture

### 5.1 High-Level Architecture

```
[React Client]  ←──HTTP/WS──→  [FastAPI Server (Python)]  ←──→  [MongoDB]
                                          ↓
                              [HuggingFace Model (local)]
                              facebook/bart-large-mnli
                              (runs inside Docker container)
```

### 5.2 Docker Compose Services

```
services:
  frontend     → React app (port 3000)
  backend      → FastAPI app (port 8000)
  mongo        → MongoDB (port 27017)
```

All three services run in the same Docker Compose network. No external AI API calls are made — the Hugging Face model is downloaded once at image build time and runs entirely inside the backend container.

### 5.3 Data Flow — Crisis Alert Lifecycle
1. User fills the crisis form; browser GPS auto-fills coordinates
2. Frontend sends POST /api/alerts to FastAPI backend
3. Backend passes the description to the local Hugging Face pipeline
4. Model returns urgency label: CRITICAL / HIGH / MEDIUM / LOW
5. Alert is saved to MongoDB with urgency label and GeoJSON coordinates
6. WebSocket broadcasts the alert to all connected volunteers within 5 km radius
7. Volunteer accepts the alert; reporter gets a real-time status update
8. Volunteer marks resolved; alert is archived

## 6. AI Urgency Classification (Hugging Face, No API)

### 6.1 Model Details

| Attribute | Value |
|---|---|
| Model | facebook/bart-large-mnli |
| Task | Zero-shot text classification |
| Library | transformers (Hugging Face) |
| Runs on | CPU inside Docker container |
| API Key Required | None |
| Internet at Runtime | Not required (model cached in image) |

### 6.2 Python Implementation

```python
from transformers import pipeline

classifier = pipeline("zero-shot-classification",
                      model="facebook/bart-large-mnli")

candidate_labels = ["critical emergency", "high urgency",
                    "medium urgency", "low urgency"]

def classify_urgency(description: str) -> str:
    result = classifier(description, candidate_labels)
    top_label = result["labels"][0]
    mapping = {
        "critical emergency": "CRITICAL",
        "high urgency": "HIGH",
        "medium urgency": "MEDIUM",
        "low urgency": "LOW"
    }
    return mapping.get(top_label, "MEDIUM")
```

### 6.3 Urgency Levels

| Level | Map Pin | Example |
|---|---|---|
| CRITICAL | Red | Person unconscious, suspected cardiac arrest |
| HIGH | Orange | Flood water entering home, elderly person alone |
| MEDIUM | Yellow | Power outage across entire block |
| LOW | Green | Stray dog injured, needs vet contact |

## 8. API Endpoints (FastAPI)

| WS | /ws/volunteer | Volunteer | WebSocket for live alert feed |

## 10. Docker Setup

### 10.1 Backend Dockerfile

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Pre-download HuggingFace model at build time
RUN python -c "from transformers import pipeline; \
    pipeline('zero-shot-classification', \
    model='facebook/bart-large-mnli')"

COPY ./app .

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### 10.2 docker-compose.yml

```yaml
version: "3.9"

services:
  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
    depends_on:
      - backend

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - MONGO_URL=mongodb://mongo:27017/neighbouraid
      - JWT_SECRET=${JWT_SECRET}
    depends_on:
      - mongo

  mongo:
    image: mongo:6
    ports:
      - "27017:27017"
    volumes:
      - mongo_data:/data/db

volumes:
  mongo_data:
```

## 11. GitHub Actions CI/CD

### 11.1 CI Pipeline — ci.yml (runs on every push)

```yaml
name: CI

on: [push, pull_request]

jobs:
  backend-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-python@v4
        with:
          python-version: "3.11"
      - run: pip install -r backend/requirements.txt
      - run: pytest backend/tests/

  frontend-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: "18"
      - run: cd frontend && npm install && npm run lint

  docker-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - run: docker compose build
```

### 11.2 Deploy Pipeline — deploy.yml (runs on merge to main)

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build and push Docker images
        run: |
          docker compose build
          docker compose push
      - name: Deploy to Render
        run: curl -X POST ${{ secrets.RENDER_DEPLOY_HOOK }}
```
